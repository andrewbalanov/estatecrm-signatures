# EstateCRM — Генератор email-подписей

Внутренний сервис для быстрого создания фирменных подписей сотрудников.
Работает полностью статически на GitHub Pages, вход только по паролю администратора.

**Сервис:** https://andrewbalanov.github.io/estatecrm-signatures/

## Возможности

- Сотрудники: имя, фамилия, должность, мобильный, email, отдел, фото.
- Обрезка фото с масштабированием до стандартного размера подписи (65×65, хранится 260×260 для Retina).
- Несколько шаблонов подписи; в каждом — выбор соцсетей (иконки + ссылки).
- Копирование подписи, оптимизированное под Outlook Windows, Outlook Mac, Mail Mac, Mail iPhone —
  с пошаговой инструкцией для каждой программы.

## Как это устроено

| Файл | Что хранит |
|---|---|
| `data/auth.json` | Соль и верификатор пароля (PBKDF2-SHA256, 600 000 итераций). Пароль по нему восстановить нельзя. |
| `data/employees.json.enc` | Данные сотрудников, зашифрованы AES-256-GCM ключом, выведенным из пароля. |
| `data/templates.json` | Настройки шаблонов: соцсети, ссылки, телефон компании, кнопка видеозвонка. |
| `js/templates.js` | Макеты шаблонов (email-safe HTML). |
| `assets/photos/` | Фотографии сотрудников (публичные — их загружают почтовые программы получателей). |
| `assets/icons/`, `assets/logo.png` | Иконки и логотип для подписей. |

Изображения в подписях **хранятся по публичным ссылкам на этом сайте**, а не встраиваются в письмо:
так подписи быстро открываются у любого получателя, не раздувают письма и не триггерят спам-фильтры
(встроенные base64-картинки режутся Gmail и Outlook). Это стандарт индустрии (так работает WiseStamp и др.).

Сохранение данных из интерфейса идёт через GitHub API: при первом входе сервис просит
fine-grained токен с доступом только к этому репозиторию (Contents: Read and write).
Токен хранится зашифрованным в браузере администратора и никуда больше не передаётся.

## Режимы

- Обычный: полный доступ (нужен токен).
- «Только просмотр»: без токена, сохранение отключено.
- Демо: добавьте `#dev` к адресу — изменения живут только в памяти браузера.

## Изменение шаблонов

Макет шаблонов меняется через Claude (Claude Code) правкой `js/templates.js`
и `data/templates.json`. Настройки соцсетей — прямо в интерфейсе, вкладка «Шаблоны».

## Служебные скрипты

```bash
# Пересоздать auth.json + пустую базу (или из seed-файла):
SIG_PASSWORD='пароль' node tools/init-data.mjs [seed.json]

# Прочитать зашифрованную базу сотрудников:
SIG_PASSWORD='пароль' node tools/decrypt-data.mjs
```

Смена пароля: `SIG_RESET=1 SIG_PASSWORD='новый' node tools/init-data.mjs seed.json`
(seed берётся из `decrypt-data.mjs` со старым паролем), затем закоммитить `data/`.
