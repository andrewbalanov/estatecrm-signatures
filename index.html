Подключение сохранения

Сохранение сейчас подключено и работает. Новый токен нужен, только если вы хотите заменить текущий — например, при увольнении сотрудника или истечении срока действия. Иначе просто нажмите «← Назад».

Чтобы сервис мог сохранять данные (у всех сотрудников), нужен токен GitHub с доступом только к этому репозиторию. Токен будет храниться в репозитории в зашифрованном виде — расшифровать его могут только вошедшие сотрудники.

Как создать токен (2 минуты)
  1. Откройте github.com/settings/personal-access-tokens/new
  2. Token name: estatecrm-signatures; Expiration: 1 год (по желанию).
  3. Repository access → «Only select repositories» → выберите estatecrm-signatures.
  4. Permissions → Repository permissions → Contents → «Read and write».
  5. Нажмите «Generate token» и скопируйте токен (начинается с github_pat_).
  6. Вставьте его в поле ниже.
